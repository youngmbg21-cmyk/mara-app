GMAIL SETUP
Google OAuth 2.0 — 5 Min Setup
Why is this needed?
Google requires apps to identify themselves before reading Gmail. This Client ID is your app's identity — it's free and takes 5 minutes to create.
Step 1 — Go to console.cloud.google.com and sign in with your Google account.
Step 2 — Click Select a project → New Project. Name it "Mara" and click Create.
Step 3 — Go to APIs & Services → Library. Search for Gmail API and click Enable.
Step 4 — Go to APIs & Services → OAuth consent screen. Choose External, fill in app name ("Mara"), your email, and click Save.
Step 5 — Go to APIs & Services → Credentials → Create Credentials → OAuth client ID. Choose Web application.
Step 6 — Under Authorized JavaScript origins, add the URL where you're running this file (e.g. http://localhost or your deployed URL). Under Authorized redirect URIs, add the same URL.
Step 7 — Click Create. Copy the Client ID (ends in .apps.googleusercontent.com) and paste it into VOYAGER.
Local file tip: If opening this HTML file directly from your computer, add http://localhost as both origin and redirect URI. Then run a simple local server: python3 -m http.server 8080 and open http://localhost:8080.
